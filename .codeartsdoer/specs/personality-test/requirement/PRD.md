# 人格测试网站 PRD

## 1. 产品概述

### 1.1 产品背景
人格测试（如MBTI、SBTI）在社交网络中具有广泛传播力和用户参与度。本项目旨在开发一个轻量级人格测试网站，让用户快速完成测试并获取人格类型结果，满足自我认知与社交分享需求。

### 1.2 目标用户
- 对自我认知和人格类型感兴趣的互联网用户
- 希望通过简短测试快速了解自身人格倾向的用户
- 有社交分享需求、愿意将测试结果分享给他人的用户

### 1.3 核心价值主张
- **快速便捷**：约3分钟完成测试，零门槛使用
- **隐私安全**：无需注册登录，不收集任何个人信息
- **结果直观**：可视化维度得分，类型描述清晰易懂
- **易于分享**：一键复制结果，便于社交传播

### 1.4 项目约束
- 开发周期：agent半天工作量
- 无持久化存储：不依赖数据库，答题进度仅存于浏览器localStorage
- 纯前端应用：无后端服务依赖
- 静态资源总大小 ≤ 500KB

## 2. 用户需求分析

### 2.1 用户画像

| 维度 | 描述 |
|------|------|
| 年龄段 | 18~35岁为主 |
| 使用场景 | 社交媒体分享、碎片时间自我探索 |
| 技术水平 | 互联网普通用户，移动端为主 |
| 核心诉求 | 快速得到有趣的人格结果并分享 |

### 2.2 用户痛点
1. 现有人格测试网站流程过长、题目过多，容易中途放弃
2. 需要注册登录才能查看结果，门槛过高
3. 结果描述过于学术化，难以理解和传播
4. 担心个人信息被收集和滥用

### 2.3 用户场景
1. 用户在社交平台看到朋友分享的人格测试结果，点击链接进入网站进行测试
2. 用户在通勤等碎片时间内，快速完成测试了解自己的人格类型
3. 用户完成测试后，将结果复制分享到微信/微博等社交平台

## 3. 功能需求

### 3.1 核心功能（P0）

#### 3.1.1 欢迎引导页
- 展示测试简介（测试主题、目的说明）
- 展示预计完成时间（约3分钟）
- 提供"开始测试"按钮
- **验收标准**：用户访问网站首页 → 可见测试简介、预计耗时、"开始测试"按钮

#### 3.1.2 答题流程
- 逐题展示，每题包含题目文本和2个选项
- 选择选项后高亮选中项，启用下一题按钮
- 实时展示答题进度（第N题/总题数）
- 必须选择当前题目的选项后才能进入下一题
- 禁止跳题
- **验收标准**：
  - 题目总数 ≥ 8，每题恰好2个选项
  - 未选择选项时点击下一题 → 系统禁止跳转并提示
  - 进入第N题 → 进度指示器显示"N/总题数"

#### 3.1.3 结果计算与展示
- 计算各维度得分（该维度极对应选项的被选次数）
- 判定人格类型（取各维度得分较高极组合）
- 得分相等时取维度定义中的第一极
- 展示人格类型名称、类型描述、各维度得分可视化
- 提供"重新测试"按钮
- **验收标准**：
  - 完成所有题目 → 各维度得分正确汇总
  - 某维度两极得分相等 → 取第一极作为结果极
  - 结果页面包含：类型名称、描述、维度可视化、重新测试按钮

#### 3.1.4 维度得分可视化
- 每个维度以可视化方式展示两极得分对比（进度条形式）
- **验收标准**：查看维度得分区域 → 每个维度均有可视化对比展示

### 3.2 重要功能（P1）

#### 3.2.1 结果分享
- 点击分享按钮，将结果摘要文本复制到剪贴板
- 分享内容包含人格类型名称与标志性描述
- 复制成功显示提示
- **验收标准**：
  - 点击复制 → 结果摘要已复制到剪贴板
  - 未点击分享 → 不触发任何分享行为

#### 3.2.2 答题进度恢复
- 浏览器刷新后从localStorage恢复答题进度
- localStorage不可用时正常答题但不提供恢复功能
- **验收标准**：
  - 答题中刷新页面 → 恢复到刷新前最后一道未答题
  - localStorage禁用 → 正常答题，刷新后需重新开始

### 3.3 优化功能（P2）

#### 3.3.1 答题动画过渡
- 题目切换时提供平滑过渡动画
- 选项选中时提供反馈动效

#### 3.3.2 结果页面视觉增强
- 人格类型搭配专属配色或图标
- 维度得分可视化采用更精美的图表样式

## 4. 信息架构

### 4.1 功能结构图

```
人格测试网站
├── 欢迎引导页
│   ├── 测试简介
│   ├── 预计耗时
│   └── 开始测试按钮
├── 答题页面
│   ├── 题目文本
│   ├── 选项A / 选项B
│   ├── 进度指示器
│   └── 下一题按钮
└── 结果页面
    ├── 人格类型名称
    ├── 类型描述
    ├── 维度得分可视化
    ├── 分享/复制按钮
    └── 重新测试按钮
```

### 4.2 页面流程图

```
欢迎页 → [点击开始测试] → 答题页(Q1→Q2→...→QN) → [完成最后一题] → 结果页
                                                                      ↓
结果页 → [点击重新测试] → 欢迎页
结果页 → [点击分享] → 复制到剪贴板 → 提示成功
```

## 5. 验收标准

| 功能 | 验收条件 | 优先级 |
|------|---------|--------|
| 欢迎页展示 | 首页可见简介、耗时、开始按钮 | P0 |
| 题目渲染 | 题目≥8，每题2选项，每维度≥2题 | P0 |
| 必答校验 | 未选择选项不可进入下一题 | P0 |
| 进度展示 | 显示当前题号/总题数 | P0 |
| 结果计算 | 各维度得分正确，类型判定正确 | P0 |
| 结果展示 | 包含类型名称、描述、维度可视化 | P0 |
| 重新测试 | 清除数据返回首页 | P0 |
| 结果分享 | 点击复制到剪贴板 | P1 |
| 进度恢复 | 刷新后恢复答题进度 | P1 |
| 题目切换动画 | 平滑过渡 | P2 |
| 结果视觉增强 | 专属配色/图标 | P2 |

## 6. 风险评估

### 6.1 技术风险
| 风险 | 概率 | 影响 | 应对策略 |
|------|------|------|---------|
| localStorage不可用 | 中 | 低 | 降级为无进度恢复，正常答题 |
| 剪贴板API兼容性 | 低 | 低 | 降级为手动选择复制 |
| 移动端布局适配问题 | 中 | 中 | 使用响应式框架，覆盖320px~1920px |

### 6.2 业务风险
| 风险 | 概率 | 影响 | 应对策略 |
|------|------|------|---------|
| 题目/类型描述质量不佳 | 中 | 高 | 类型描述需精心编写，兼顾专业性与趣味性 |
| 测试结果缺乏分享传播力 | 中 | 中 | 结果描述简短有趣，便于社交传播 |
