# 人格测试网站 PRD（扩展版）

## 1. 产品概述

### 1.1 产品背景
人格测试（如MBTI、大五人格、九型人格）在社交网络中具有广泛传播力和用户参与度。本项目旨在开发一个多主题人格测试平台，让用户可以选择不同测试体系、完成测试并获取人格类型结果，同时提供类型百科、类型匹配、历史记录等深度功能，满足自我认知与社交分享需求。

### 1.2 目标用户
- 对自我认知和人格类型感兴趣的互联网用户
- 希望通过简短测试快速了解自身人格倾向的用户
- 有社交分享需求、愿意将测试结果分享给他人的用户
- 对人格类型理论感兴趣，希望深入了解各类型特征的用户
- 好奇自己与他人人格匹配度的用户

### 1.3 核心价值主张
- **多主题选择**：MBTI、大五人格、九型人格三大体系任选
- **快速便捷**：约3~5分钟完成测试，零门槛使用
- **深度认知**：类型百科提供详尽人格解析，匹配功能揭示人际相容性
- **隐私安全**：无需注册登录，不收集任何个人信息
- **结果直观**：专属配色+精美可视化维度得分
- **易于分享**：一键复制结果，便于社交传播
- **体验精致**：明暗主题、流畅动效、响应式布局

### 1.4 项目约束
- 开发周期：agent约3小时工作量
- 无持久化存储：不依赖数据库，答题进度与历史记录仅存于浏览器localStorage
- 纯前端应用：无后端服务依赖
- 静态资源总大小 ≤ 800KB
- 技术栈：Vue 3 + Vite + Vue Router + Pinia + CSS/SCSS + TypeScript

## 2. 用户需求分析

### 2.1 用户画像

| 维度 | 描述 |
|------|------|
| 年龄段 | 18~35岁为主 |
| 使用场景 | 社交媒体分享、碎片时间自我探索、深度了解人格理论 |
| 技术水平 | 互联网普通用户，移动端为主 |
| 核心诉求 | 快速得到有趣的人格结果并分享，深入了解自己与他人 |

### 2.2 用户痛点
1. 现有人格测试网站通常只有单一测试体系，无法对比不同体系的结论
2. 测试后对类型描述过于简短或学术化，难以深入理解
3. 不知道自己与朋友/伴侣的类型匹配度，缺乏人际互动视角
4. 无法回顾过往测试结果，难以观察自身变化
5. 页面视觉单调，缺乏个性化与沉浸感
6. 担心个人信息被收集和滥用

### 2.3 用户场景
1. 用户在社交平台看到朋友分享的人格测试结果，点击链接进入网站选择感兴趣的测试主题进行测试
2. 用户在通勤等碎片时间内，快速完成MBTI测试了解自己的人格类型
3. 用户完成测试后，进入类型百科深入了解自己类型的特征、优势和职业倾向
4. 用户想知道自己(INTJ)和朋友(ENFP)的匹配度，进入匹配分析功能
5. 用户想回顾自己过去几次测试的结果，打开历史记录查看
6. 用户在夜间使用，切换至暗色模式保护视力
7. 用户完成测试后，将结果复制分享到微信/微博等社交平台

## 3. 功能需求

### 3.1 核心功能（P0）

#### 3.1.1 首页与测试主题选择
- 展示网站标题与简介
- 展示可用测试主题列表（MBTI、大五人格、九型人格），每项含主题名称、描述、题数、预计耗时
- 提供"开始测试"按钮进入对应主题答题流程
- 导航栏提供"首页"、"类型百科"、"历史记录"入口
- **验收标准**：首页可见网站标题、3个测试主题卡片、导航栏；点击主题"开始测试"→进入该主题答题页

#### 3.1.2 答题流程
- 逐题展示，每题包含题目文本和2个选项
- 选择选项后高亮选中项，启用下一题按钮
- 实时展示答题进度（第N题/总题数）和当前测试主题名称
- 必须选择当前题目的选项后才能进入下一题
- 禁止跳题
- **验收标准**：
  - 各主题题目总数 ≥ 8，每题恰好2个选项
  - 未选择选项时点击下一题 → 系统禁止跳转并提示
  - 进入第N题 → 进度指示器显示"N/总题数"
  - 答题页顶部显示当前主题名称

#### 3.1.3 结果计算与展示
- 计算各维度得分（该维度极对应选项的被选次数）
- 判定人格类型（取各维度得分较高极组合）
- 得分相等时取维度定义中的第一极
- 展示人格类型名称、类型描述、各维度得分可视化
- 应用该类型的专属配色（主色、辅助色、背景色）
- 提供"重新测试"按钮
- 自动保存结果至历史记录
- **验收标准**：
  - 完成所有题目 → 各维度得分正确汇总
  - 某维度两极得分相等 → 取第一极作为结果极
  - 结果页面包含：类型名称、描述、维度可视化、专属配色、重新测试按钮
  - localStorage中存在本次测试历史记录

#### 3.1.4 类型百科
- 提供测试主题选择器，可切换不同主题的百科内容
- 以卡片列表展示当前主题下所有人格类型，每张卡片含类型名称、核心特质标签、专属配色
- 点击卡片展示类型详情：名称、别名、核心特质、优势、劣势、典型职业、最佳匹配类型
- 提供搜索功能，按名称或核心特质筛选类型
- **验收标准**：
  - 进入百科页面 → 显示主题选择器和类型卡片列表
  - 切换主题 → 卡片列表更新为新主题类型
  - 搜索关键词 → 仅显示匹配的类型；无匹配 → 显示"未找到匹配的人格类型"
  - 点击卡片 → 展示完整详情

#### 3.1.5 维度得分可视化
- 每个维度以进度条形式展示两极得分对比
- 使用当前类型的专属配色
- **验收标准**：查看维度得分区域 → 每个维度均有可视化对比展示且配色与类型专属配色一致

### 3.2 重要功能（P1）

#### 3.2.1 结果分享
- 点击分享按钮，将结果摘要文本（含主题名称、类型名称与标志性描述）复制到剪贴板
- 复制成功显示提示
- **验收标准**：
  - 点击复制 → 结果摘要已复制到剪贴板
  - 未点击分享 → 不触发任何分享行为

#### 3.2.2 答题进度恢复
- 浏览器刷新后从localStorage恢复答题进度
- localStorage不可用时正常答题但不提供恢复功能
- **验收标准**：
  - 答题中刷新页面 → 恢复到刷新前最后一道未答题
  - localStorage禁用 → 正常答题，刷新后需重新开始

#### 3.2.3 类型匹配
- 在结果页或百科页可进入匹配分析界面
- 一端预选用户类型，另一端可选择同主题下所有类型
- 计算并展示匹配度百分比（环形图或进度条）与匹配分析描述
- **验收标准**：
  - 进入匹配界面 → 一端预选用户类型，另一端为类型选择器
  - 选择类型B → 展示匹配度百分比和分析描述
  - A匹配B与B匹配A的匹配度相同

#### 3.2.4 历史记录
- 按时间倒序展示所有历史记录，每条含测试时间、主题、类型名称、配色标记
- 点击记录查看该次测试完整结果
- 提供删除单条记录和清空所有记录功能
- 无记录时展示空状态提示和"去测试"按钮
- 最多保存20条，超出自动删除最旧记录
- **验收标准**：
  - 有历史记录 → 按时间倒序列出，每条含时间、主题、类型名
  - 无历史记录 → 展示"暂无测试记录"提示和"去测试"按钮
  - 点击删除 → 记录移除；点击清空 → 所有记录清除

#### 3.2.5 明暗主题切换
- 提供主题切换按钮（亮色↔暗色）
- 首次访问跟随系统`prefers-color-scheme`
- 用户选择持久化至localStorage
- CSS变量全局应用
- **验收标准**：
  - 首次访问+系统暗色 → 网站默认暗色
  - 点击切换 → 主题切换且刷新后保持
  - 切换后所有页面元素配色同步更新

#### 3.2.6 非法页面访问处理
- 未完成测试时直接访问结果页面应重定向至首页
- **验收标准**：非法访问结果页→重定向至首页

### 3.3 优化功能（P2）

#### 3.3.1 页面过渡动画
- 题目切换：滑出/滑入过渡（≤400ms）
- 选项选中：微交互动效（≤200ms）
- 页面路由：淡入淡出（≤300ms）
- 支持`prefers-reduced-motion`减弱动效偏好

#### 3.3.2 结果页面视觉增强
- 人格类型专属配色与图标
- 维度得分可视化精美化（渐变进度条、动画填充）
- 匹配度环形进度图

## 6. 风险评估

### 6.1 技术风险
| 风险 | 概率 | 影响 | 应对策略 |
|------|------|------|---------|
| localStorage不可用 | 中 | 中 | 历史记录和进度恢复降级为不可用，核心测试流程不受影响 |
| 剪贴板API兼容性 | 低 | 低 | 降级为手动选择复制 |
| 移动端布局适配问题 | 中 | 中 | 使用响应式框架，覆盖320px~1920px |
| 多主题数据量过大 | 中 | 中 | 控制百科文案长度，懒加载非当前主题数据 |
| 匹配度规则设计复杂度 | 中 | 低 | 基于维度极一致性/互补性简化规则，配置化定义 |
| 静态资源超800KB | 低 | 中 | 精简文案数据，使用代码分割和懒加载 |

### 6.2 业务风险
| 风险 | 概率 | 影响 | 应对策略 |
|------|------|------|---------|
| 题目/类型描述质量不佳 | 中 | 高 | 三大主题的题目与描述需精心编写，兼顾专业性与趣味性 |
| 测试结果缺乏分享传播力 | 中 | 中 | 结果描述简短有趣，专属配色增强视觉传播力 |
| 多主题增加用户选择困难 | 低 | 中 | 首页设计突出推荐主题(MBTI)，其他主题自然展示 |
| 匹配度缺乏科学依据 | 中 | 低 | 明确标注匹配度为娱乐参考，基于简化规则计算 |
| 历史记录可能被用户清除 | 低 | 低 | 依赖localStorage，清除属用户主动行为，无需特殊处理 |

### 6.3 工作量风险
| 风险 | 概率 | 影响 | 应对策略 |
|------|------|------|---------|
| 3小时工作量不足以实现全部P1功能 | 中 | 中 | 严格按P0>P1>P2优先级交付，P1中历史记录可降级至P2 |
| 三套主题数据编写耗时过长 | 中 | 中 | 九型人格题目精简至8题，大五人格使用简化版10题 |
