# 需求文档

## 1. 概述

本文档基于spec.md需求规格，对人格测试网站的完整需求进行系统化整理，涵盖功能需求、非功能需求与约束条件。

## 2. 功能需求

### FR-01 欢迎与引导

| 属性 | 内容 |
|------|------|
| 需求ID | FR-01 |
| 需求名称 | 欢迎与引导 |
| 优先级 | P0 |
| 描述 | 网站首页必须展示测试简介、预计完成时间和开始测试入口 |
| 前置条件 | 无 |
| 触发条件 | 用户访问网站 |
| 正常流程 | 1. 系统加载首页 2. 展示测试简介 3. 展示预计耗时（如"约3分钟"） 4. 展示"开始测试"按钮 |
| 验收标准 | 首页可见：测试简介、预计耗时、"开始测试"按钮 |

### FR-02 答题流程

| 属性 | 内容 |
|------|------|
| 需求ID | FR-02 |
| 需求名称 | 答题流程 |
| 优先级 | P0 |
| 描述 | 用户逐题完成问卷，每题选择一个选项后进入下一题，直至完成所有题目 |
| 前置条件 | 用户点击"开始测试" |
| 触发条件 | 进入答题页面 |
| 正常流程 | 1. 展示第1题（题目文本+2选项+进度1/N） 2. 用户选择选项 3. 高亮选中项 4. 用户点击下一题 5. 展示下一题，进度更新 6. 重复至最后一题完成 |
| 业务规则 | BR-01: 题目总数 ≥ 8 |
|  | BR-02: 每题恰好2个选项 |
|  | BR-03: 必须选择选项后才可进入下一题 |
|  | BR-04: 禁止跳题 |
|  | BR-05: 每个维度至少2道题 |
| 验收标准 | 1. 题目≥8，每题2选项 2. 未选择不可下一题 3. 进度实时更新 |

### FR-03 结果计算

| 属性 | 内容 |
|------|------|
| 需求ID | FR-03 |
| 需求名称 | 测试结果计算 |
| 优先级 | P0 |
| 描述 | 根据答题数据计算各维度得分并判定人格类型 |
| 前置条件 | 用户完成所有题目 |
| 触发条件 | 最后一题提交 |
| 正常流程 | 1. 汇总各维度关联题目的选项选择 2. 计算各维度两极得分 3. 各维度取高分极组合为类型 4. 渲染结果页面 |
| 业务规则 | BR-06: 维度得分 = 该极对应选项被选次数 |
|  | BR-07: 人格类型 = 各维度高分极组合 |
|  | BR-08: 得分相等时取维度定义的第一极 |
| 验收标准 | 1. 各维度得分正确汇总 2. 类型判定正确 3. 计算耗时≤500ms |

### FR-04 结果展示

| 属性 | 内容 |
|------|------|
| 需求ID | FR-04 |
| 需求名称 | 测试结果展示 |
| 优先级 | P0 |
| 描述 | 展示人格类型名称、描述、维度得分可视化和重新测试入口 |
| 前置条件 | 结果计算完成 |
| 触发条件 | 进入结果页面 |
| 正常流程 | 1. 展示人格类型名称 2. 展示类型描述 3. 展示各维度可视化得分 4. 展示分享按钮 5. 展示重新测试按钮 |
| 验收标准 | 结果页包含：类型名称、描述、维度可视化、重新测试按钮 |

### FR-05 结果分享

| 属性 | 内容 |
|------|------|
| 需求ID | FR-05 |
| 需求名称 | 结果分享 |
| 优先级 | P1 |
| 描述 | 用户可将测试结果摘要复制到剪贴板 |
| 前置条件 | 结果页面已展示 |
| 触发条件 | 用户点击分享/复制按钮 |
| 正常流程 | 1. 生成结果摘要文本（类型名称+标志性描述） 2. 写入剪贴板 3. 显示复制成功提示 |
| 异常流程 | 剪贴板写入失败 → 展示可手动选择的文本区域，提示"请手动选择并复制" |
| 业务规则 | BR-09: 分享内容包含类型名称和描述 |
|  | BR-10: 禁止未经用户操作自动分享 |
| 验收标准 | 1. 点击复制→摘要到剪贴板 2. 未点击不触发分享 |

### FR-06 答题进度恢复

| 属性 | 内容 |
|------|------|
| 需求ID | FR-06 |
| 需求名称 | 答题进度恢复 |
| 优先级 | P1 |
| 描述 | 浏览器刷新后从localStorage恢复答题进度 |
| 前置条件 | 用户已答部分题目且未完成 |
| 触发条件 | 浏览器刷新 |
| 正常流程 | 1. 检测localStorage中存在进度数据 2. 恢复已答题目记录 3. 定位到刷新前未答题目 4. 展示该题 |
| 异常流程 | localStorage不可用 → 正常答题，不提供恢复功能 |
| 验收标准 | 1. 刷新后恢复到未答题目 2. 已答题目无需重答 3. localStorage不可用时正常使用 |

### FR-07 非法页面访问处理

| 属性 | 内容 |
|------|------|
| 需求ID | FR-07 |
| 需求名称 | 非法页面访问处理 |
| 优先级 | P1 |
| 描述 | 未完成测试时直接访问结果页面应重定向至首页 |
| 前置条件 | 无 |
| 触发条件 | 通过URL直接访问结果页面且无完整答题数据 |
| 正常流程 | 1. 检测答题数据不完整 2. 重定向至首页 3. 可选展示提示"请先完成测试" |
| 验收标准 | 非法访问结果页→重定向至首页 |

## 3. 非功能需求

### NFR-01 性能

| 编号 | 需求 | 指标 |
|------|------|------|
| NFR-01-01 | 首屏加载时间 | ≤ 2秒 |
| NFR-01-02 | 单题切换响应 | ≤ 200毫秒 |
| NFR-01-03 | 结果计算与渲染 | ≤ 500毫秒 |
| NFR-01-04 | 静态资源总大小 | ≤ 500KB |

### NFR-02 安全性

| 编号 | 需求 | 说明 |
|------|------|------|
| NFR-02-01 | 禁止收集个人信息 | 不收集姓名、邮箱、手机号等 |
| NFR-02-02 | 禁止外部数据传输 | 不向第三方发送答题数据 |
| NFR-02-03 | 禁止第三方追踪 | 不使用Google Analytics等追踪脚本 |

### NFR-03 兼容性

| 编号 | 需求 | 说明 |
|------|------|------|
| NFR-03-01 | 浏览器支持 | Chrome、Firefox、Safari、Edge最近2个主版本 |
| NFR-03-02 | 移动端适配 | 响应式布局，320px~1920px |
| NFR-03-03 | 纯前端 | 不依赖服务端运行时 |

### NFR-04 可维护性

| 编号 | 需求 | 说明 |
|------|------|------|
| NFR-04-01 | 维度配置化 | 维度数量与题目以配置定义，便于扩展 |
| NFR-04-02 | 文案分离 | 类型描述与代码逻辑分离，便于独立更新 |

### NFR-05 可靠性

| 编号 | 需求 | 说明 |
|------|------|------|
| NFR-05-01 | 站点可用性 | ≥ 99.5%（静态站点依赖CDN） |
| NFR-05-02 | 进度恢复 | 刷新后答题进度可恢复 |

## 4. 数据约束

### DC-01 人格维度
- 维度标识：英文大写字母，1~3字符
- 维度名称：非空，≤10字符
- 极A/极B名称：非空，≤10字符
- 维度数量：2~6个

### DC-02 测试题目
- 题目文本：非空，≤100字符
- 选项文本：非空，≤50字符
- 每题关联1个维度，选项分别对应两极

### DC-03 人格类型
- 类型标识：各维度结果极标识拼接
- 类型名称：非空，≤20字符
- 类型描述：非空，≤500字符
- 类型数量：2^维度数

### DC-04 答题记录
- 仅存于localStorage，不传输至服务端
- 浏览器会话有效期内有效
- 完成测试后可选清除
