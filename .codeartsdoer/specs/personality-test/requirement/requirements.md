# 需求文档（扩展版）

## 1. 概述

本文档基于spec.md需求规格，对人格测试网站（扩展版）的完整需求进行系统化整理，涵盖功能需求、非功能需求与约束条件。扩展版新增多测试主题、类型百科、类型匹配、历史记录、明暗主题、交互动效等功能模块。

## 2. 功能需求

### FR-01 首页与测试主题选择

| 属性 | 内容 |
|------|------|
| 需求ID | FR-01 |
| 需求名称 | 首页与测试主题选择 |
| 优先级 | P0 |
| 描述 | 网站首页必须展示网站标题、简介、可用测试主题列表和导航入口 |
| 前置条件 | 无 |
| 触发条件 | 用户访问网站 |
| 正常流程 | 1. 系统加载首页 2. 展示网站标题与简介 3. 展示测试主题列表（每个主题含名称、描述、题数、预计耗时、开始测试按钮） 4. 展示导航栏（首页、类型百科、历史记录） |
| 业务规则 | BR-01: 测试主题数量 ≥ 3 |
|  | BR-02: 必须包含MBTI、大五人格、九型人格三个主题 |
| 验收标准 | 1. 首页可见3个主题卡片 2. 点击主题"开始测试"→进入该主题答题页 3. 导航栏包含首页、百科、历史入口 |

### FR-02 答题流程

| 属性 | 内容 |
|------|------|
| 需求ID | FR-02 |
| 需求名称 | 答题流程 |
| 优先级 | P0 |

| 触发条件 | 进入答题页面 |
| 正常流程 | 1. 设置当前测试主题 2. 展示第1题（主题名称+题目文本+2选项+进度1/N） 3. 用户选择选项 4. 高亮选中项 5. 用户点击下一题 6. 展示下一题，进度更新 7. 重复至最后一题完成 |
| 业务规则 | BR-03: 各主题题目总数 ≥ 8 |
|  | BR-04: 每题恰好2个选项 |
|  | BR-05: 必须选择选项后才可进入下一题 |
|  | BR-06: 禁止跳题 |
|  | BR-07: 每个维度至少2道题 |
|  | BR-08: 答题页顶部展示当前测试主题名称 |
| 验收标准 | 1. 各主题题目≥8，每题2选项 2. 未选择不可下一题 3. 进度实时更新 4. 顶部显示主题名称 |

### FR-03 结果计算

| 属性 | 内容 |
|------|------|
| 需求ID | FR-03 |
| 需求名称 | 测试结果计算 |
| 优先级 | P0 |
| 描述 | 根据答题数据计算各维度得分并判定人格类型，自动保存至历史记录 |
| 前置条件 | 用户完成所有题目 |
| 触发条件 | 最后一题提交 |
| 正常流程 | 1. 汇总各维度关联题目的选项选择 2. 计算各维度两极得分 3. 各维度取高分极组合为类型 4. 自动保存结果至历史记录 5. 渲染结果页面 |
| 业务规则 | BR-09: 维度得分 = 该极对应选项被选次数 |
|  | BR-10: 人格类型 = 各维度高分极组合 |
|  | BR-11: 得分相等时取维度定义的第一极 |
|  | BR-12: 结果完成后自动保存至历史记录 |
| 验收标准 | 1. 各维度得分正确汇总 2. 类型判定正确 3. 计算耗时≤500ms 4. localStorage含历史记录 |

## 4. 数据约束

### DC-01 测试主题
- 主题标识：小写英文字母与连字符，2~20字符
- 主题名称：非空，≤20字符
- 主题描述：非空，≤200字符
- 预计完成时间：格式"约X分钟"
- 主题数量：≥3

### DC-02 人格维度
- 维度标识：英文大写字母，1~3字符
- 维度名称：非空，≤10字符
- 极A/极B名称：非空，≤10字符
- 维度数量：每主题2~6个

### DC-03 测试题目
- 题目文本：非空，≤100字符
- 选项文本：非空，≤50字符
- 每题关联1个维度，选项分别对应两极

### DC-04 人格类型
- 类型标识：各维度结果极标识拼接
- 类型名称：非空，≤20字符
- 类型描述：非空，≤500字符
- 类型数量：2^维度数
- 专属配色：主色、辅助色、背景色（HEX格式）
- 核心特质标签：2~5项，每项≤6字符

### DC-05 类型百科内容
- 别名：≤30字符
- 核心特质：非空，≤200字符
- 优势：2~5项，每项≤50字符
- 劣势：2~5项，每项≤50字符
- 典型职业：2~5项，每项≤20字符
- 最佳匹配类型：类型标识

### DC-06 匹配度规则
- 匹配度范围：0%~100%，整数
- 对称性：A↔B = B↔A
- 自匹配度：100%
- 分析描述：≤200字符

### DC-07 历史记录
- 仅存于localStorage，不传输至服务端
- 最多20条，FIFO淘汰
- 单条记录≤2KB
- 含测试时间(ISO 8601)、主题标识、类型标识、维度得分快照

### DC-08 答题记录
- 仅存于localStorage
- 含当前主题标识、当前题号、已答题目列表
- 完成测试后清除进度

### DC-09 明暗主题
- 主题值："light"或"dark"
- 存于localStorage的theme字段
- 默认跟随prefers-color-scheme，无法检测时默认"light"
